# <a name="explore-the-azure-and-cloud-products-benefits-area-in-partner-center"></a>Erkunden Sie den Bereich "Vorteile von Azure und Cloud-Produkten" im Partner Center

**Geeignete Rollen**

- Globaler Administrator
- MPN-Partneradministrator

In diesem Artikel werden die spezifischen Azure-und cloudanwendungen beschrieben, die von Partnern innerhalb von Partner Center verfügbar sind. Außerdem erfahren Sie, wie Sie bestimmte Arten von Azure-und cloudabonnements aktivieren.
 
## <a name="overview"></a>Übersicht

Als Partner im Microsoft Partner Network haben Sie möglicherweise Zugriff auf viele technische und Markt Einführungs Vorteile für verschiedene Bereiche. Wenn Sie eine der oben genannten Rollen im Partner Center zugewiesen haben, können Sie viele dieser Vorteile direkt über das Partner Center-Dashboard untersuchen.

> [!NOTE]
> Um auf den Bereich mit den Vorteilen von Partner Center zuzugreifen, müssen Sie sich nur beim Partner Center- [Dashboard](https://partner.microsoft.com/dashboard) anmelden und im linken Navigationsmenü **MPN** auswählen. Wählen Sie dann im Menü die Option **Vorteile** aus, gefolgt von der Option bestimmte Vorteile oder den Bereich, den Sie untersuchen möchten.

Eine der Vorteile, die Ihnen möglicherweise für Sie zur Verfügung stehen, ist **Azure & Cloud-Produkte**. Wenn Sie **Azure & Cloud** im **Partner Center-** Vorteils Menü auswählen, wird die Seite **Azure & Cloud-Produkte** angezeigt.

Auf dieser Seite werden die folgenden Details angezeigt:

- Es stehen verschiedene Vorteile von Azure & Cloud-Produkt Abonnements zur Verfügung, unabhängig davon, ob Lizenz basierte Abonnements oder nutzungsbasierte Abonnements vorhanden sind.
- Zugewiesene Lizenz Menge für jedes Produkt
- Der Angebotstyp, der Ihnen ein bestimmtes Vorteil bereitgestellt hat.
- Der vorganstyp (unabhängig davon, ob er von einem Kern oder inkrementellen
- Das Datum, an dem ein bestimmter Vorteil abläuft

:::image type="content" source="images/benefits/mpn-benefits-azure-cloud-1.png" alt-text="Die Seite mit den Azure-und cloudprodukten, die Ihnen verfügbare Abonnement Vorteile bietet." border="false":::

## <a name="usage-based-vs-license-based-subscriptions"></a>Nutzungsbasierte und Lizenz basierte Abonnements

Auf der Seite **Azure & Cloud-Produkte** werden zwei Arten von Abonnement Vorteilen aufgelistet, die Ihnen bereitgestellt werden:
- Nutzungsbasierte Abonnements (Microsoft Azure Guthaben) 
- Lizenz basierte Abonnements (z. b. Office 365)

### <a name="usage-based-subscriptions-with-two-types-of-microsoft-azure-credit"></a>Nutzungsbasierte Abonnements mit zwei Typen von Microsoft Azure Guthaben

Abhängig von den aktiven angeboten, die für Ihre Microsoft Partner Network Mitgliedschaft verfügbar sind, gibt es zwei Arten von Verwendungs basierten Abonnements, die Sie erhalten können. Zu diesen aktiven Angeboten können Microsoft Action Pack, eine Silver-Kompetenz oder eine Gold-Kompetenz gehören. 

Die zwei Arten von Verwendungs basierten Abonnements, die für diese Angebote bereitgestellt werden können, sind die *monatliche Azure-Gutschrift* und/oder die *Azure Bulk-Gutschrift (jährlich)*:

- Bereitstellung **für monatliche Azure-Gutschrift:** Dies könnte etwa etwa 100 USD/Monat (oder gleichwertig in der lokalen Währung) erscheinen.<br/>
*und/oder*
- Bereitstellung **für Azure Bulk-Gutschrift (jährlich):** Dies könnte etwa wie folgt aussehen: $6000/Jahr, $12000/Year und/oder $13000/Year (oder jeweils gleichwertig in der lokalen Währung).

Zum Aktivieren eines Microsoft Azure Abonnements in Partner Center müssen Sie einem Benutzer innerhalb des Unternehmens (oder eines Gast Benutzers) Zugriff zuweisen. Der Benutzer muss bereits über einen Benutzereintrag verfügen, der in Ihrem Azure Active Directory Mandanten erstellt wurde. Dann führen Sie folgende Schritte aus:

1. Wenn Sie sicher sind, dass der Benutzer über einen Benutzereintrag in Ihrem Azure AD-Mandanten verfügt, melden Sie sich beim Partner Center- [Dashboard](https://partner.microsoft.com/dashboard)an.

1. Wählen Sie im linken Navigationsmenü **MPN** aus, gefolgt von den **Vorteilen** und dann **Azure und Cloud**.

1. Klicken Sie in der rechten Ecke auf den Pfeil nach unten, und klicken Sie auf die Spalte **Details** für das nutzungsbasierte Abonnement, das Sie aktivieren möchten. 

1. Suchen Sie den Bereich **Benutzer Zuweisung** auf der Seite Partner Center **Azure-und cloudprodukte** (unten). Wählen Sie dann die Option **Benutzer zuweisen** aus, um diesem Benutzer den gewünschten nutzungsbasierten Abonnement Vorteil zuzuweisen. 

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-2.png" alt-text="Azure-und cloudprodukte-Seite mit Benutzer Zuweisungs Bereich." border="false":::

- Wenn Sie eine monatliche Azure-Gutschrift von etwa $100 pro Monat zuweisen (oder gleichwertig in der lokalen Währung), können Sie Benutzer zu diesem Abonnement jederzeit hinzufügen oder entfernen.

- Wenn Sie die Azure Bulk-Gutschrift (jährlich) zuweisen, beachten Sie, dass die Option **Entfernen** nicht in Partner Center verfügbar ist. Stattdessen müssen Sie das Massen Guthaben während des Registrierungs Jahrs an eine andere Person übertragen. Das Azure-Massen Abonnement (jährlich) kann auch nicht mit anderen Lizenzierungs-, Zahlungs-oder Kredit angeboten (z. b. Microsoft Enterprise Agreement, cloudlösungsanbieter, Azure in Open, Azure Pass, Microsoft Action Pack usw.) kombiniert werden.

### <a name="license-based-subscriptions"></a>Lizenzbasierte Abonnements

Im Abschnitt Lizenz basierte Abonnements der Seite Azure-und cloudprodukte finden Sie alle cloudprodukte, für die eine Product Key zur Aktivierung erforderlich ist, z. b. Microsoft O365, Microsoft Dynamics, Microsoft InTune, Azure AD Basic, Power BI usw. Diese Produkte werden basierend auf dem Typ des aktiven Angebots angeboten, das Ihnen als Partner im Microsoft Partner Network zur Verfügung steht. Solche Angebote können Microsoft-Action Pack, Silver-Kompetenz Mitgliedschaft oder Gold-Kompetenz Mitgliedschaft enthalten.

So aktivieren Sie ein Lizenz basiertes Abonnement: 

1. Suchen Sie auf der Seite Azure-und cloudprodukte nach dem jeweiligen Abonnement. 

1. Wählen Sie auf der rechten Seite den Dropdown Pfeil aus, und klicken Sie auf die Spalte **Details** für dieses Abonnement. 

1. Lesen Sie die Anweisungen auf der nachfolgenden Seite. Auf diese Weise erfahren Sie mehr darüber, wie die Schlüssel eingelöst werden. 

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-3.png" alt-text="Seite &quot;Azure-und cloudprodukte&quot; mit Details anzeigen." border="false":::


1. Wenn Sie bereit sind, den Schlüssel einzulösen, wählen Sie die Option Exemplarische Vorgehensweise für die **Aktivierung** unten auf dieser Seite aus.
   - Wenn Sie ein neues Abonnement aktivieren möchten oder Ihr vorhandenes Abonnement deaktiviert/abgelaufen ist, öffnen Sie einen **privaten oder inkognito-Browser**, kopieren Sie den folgenden Link, und fügen Sie ihn ein, und folgen Sie den Anweisungen:

      [https://signup.microsoft.com/productkeystart](https://signup.microsoft.com/productkeystart)
      
   - Wenn Sie das Enddatum erweitern oder einer vorhandenen Abonnement Kopie weitere Arbeitsplätze/Lizenzen hinzufügen möchten, fügen Sie den folgenden Link in ein **privates oder inkognito-Browserfenster** ein, und folgen Sie den Anweisungen:
 
      [https://admin.microsoft.com/AdminPortal/Home#/subscriptions](https://admin.microsoft.com/AdminPortal/Home#/subscriptions)

> [!Note]
> Nachdem ein Schlüssel eingelöst wurde, kann derselbe Schlüssel nicht ein zweites Mal eingelöst werden. Beachten Sie auch die folgenden wichtigen Punkte:
>
> - Vergewissern Sie sich, dass Sie weitere Lizenzen (Arbeitsplätze) hinzufügen müssen, wie viele sind und ob das Enddatum erweitert werden soll. Wenn Sie weitere Lizenzen (Arbeitsplätze) hinzufügen und das Enddatum **nicht** verlängern, bietet Ihnen das System weniger als 12 Monate aktives Abonnement. 
>
> - Stellen Sie sicher, dass Sie mit den richtigen Anmelde Informationen angemeldet sind und die Schlüssel im richtigen Azure Active Directory (AAD)-Mandanten aktivieren. Melden Sie sich immer mit einem privaten/inkognito-Browserfenster an, um sicherzustellen, dass Sie Schlüssel auf dem richtigen Mandanten aktivieren.
>
> - Das Ablaufdatum des Vorteils, das in Microsoft Partner Center angezeigt wird, gibt das letzte Datum an, an dem die Nutzungsrechte bei diesem Vorteil Dieses Datum unterscheidet sich möglicherweise vom Enddatum des Abonnements, das im M365 Admin Center-Portal aufgeführt ist. Dies liegt möglicherweise an mehreren Faktoren, z. b. dem Aktivierungsdatum des Vorteils, der Anzahl der Lizenzen, die dem Schlüssel zugeordnet sind, und den Lizenzen, die im Verwaltungs Portal hinzugefügt oder erweitert wurden.

## <a name="extend-the-end-date-or-add-more-licenses-to-an-existing-subscription"></a>Erweitern Sie das Enddatum, oder fügen Sie einem vorhandenen Abonnement weitere Lizenzen hinzu.

Führen Sie die folgenden Schritte aus, um weitere Lizenzen hinzuzufügen oder das Enddatum für ein vorhandenes Abonnement zu verlängern:

1. Kopieren Sie den folgenden Link, und fügen Sie ihn in ein **privates oder inkognito-Browserfenster** ein.

   [https://admin.microsoft.com/AdminPortal/Home#/subscriptions](https://admin.microsoft.com/AdminPortal/Home#/subscriptions)

1. Wenn Sie sich beim **M365 Admin Center-Portal** anmelden, stellen Sie sicher, dass Sie sich mit dem richtigen Azure Active Directory Mandanten anmelden. Wählen Sie dann in der Liste das betreffende Cloud-Abonnement aus, und führen Sie einen der folgenden Schritte aus (um das Abonnement zu erneuern oder weitere Lizenzen hinzuzufügen):

   - Wenn Sie das vorhandene Abonnement erneuern möchten, wählen Sie im Bereich Lizenz die Option **Enddatum erweitern** aus. Durch das Erweitern des Enddatums wird die Dauer des Abonnements verlängert, während gleichzeitig die Anzahl der Lizenzen/Plätze unverändert bleibt. 

   - Wenn Sie einem vorhandenen Abonnement weitere Lizenzen (Arbeitsplätze) hinzufügen möchten, wählen Sie im Bereich Lizenz die Option **Lizenzen hinzufügen** aus. Stellen Sie sicher, dass Sie keine **Lizenzen hinzufügen** auswählen, wenn Sie auch das vorhandene Abonnement erneuern möchten. Wenn **Sie Lizenzen hinzufügen** auswählen, wird die Abonnementdauer reduziert und die Anzahl von Lizenzen im Abonnement erhöht.

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-4.png" alt-text="Die Seite &quot;Azure-und cloudprodukte&quot; mit dem Hinzufügen von Lizenzen." border="false":::

1. Wählen Sie im Seitenbereich die Option **Product Key einlösen** aus. 

1. Geben Sie den Product Key ein, **und wählen Sie** dann überprüfen aus, um die Schlüssel Details zu 

1. Wählen Sie **einlösen** aus, um den Vorgang abzuschließen.

   > [!NOTE]
   > Jede Product Key bietet Benutzern 12 Monate lang einen Dienst ab dem Datum, an dem Sie die Product Key einlösen. Wenn Ihre vorhandenen Benutzer jedoch ein frühes Erneuerungsdatum aufweisen, werden zwei Datumsangaben übermittelt, um die Verwaltung Ihrer Abonnements auf der Grundlage eines Enddatums für alle Benutzer zu vereinfachen. Während der Berechnung wird keine Zeit hinzugefügt oder verloren. Die Berechnung für den Mittelwert von zwei Datumsangaben ist:
   >
   > **Berechnung:** \[ A (anfängliche Benutzer) × B (verbleibende Zeit für Abonnement) \+ C (neue Benutzer) x (12 Monate), \] Gesamtzahl der Benutzer = neu verbleibende Zeit.

## <a name="activate-a-subscription-for-the-first-time"></a>Aktivieren eines Abonnements zum ersten Mal

Führen Sie diese Schritte aus, wenn Sie ein Abonnement zum ersten Mal aktivieren. Sie können diese Schritte auch ausführen, wenn sich Ihr Abonnement im Status "deaktiviert" oder "abgelaufen" befindet:

1. Kopieren Sie den folgenden Link, und fügen Sie ihn in ein **privates oder inkognito-Browserfenster** ein.
   
   [https://signup.microsoft.com/productkeystart](https://signup.microsoft.com/productkeystart)
1. Stellen Sie sicher, dass Sie sich beim richtigen Azure Active Directory Mandanten Konto angemeldet haben. Nachdem Sie angemeldet sind, überprüfen Sie die Details des Mandanten.

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-5.png" alt-text="Die Seite &quot;Azure-und cloudprodukte&quot;, die anzeigt, wo der Schlüssel" border="false":::

1. Geben Sie den 25-stelligen Schlüssel aus Partner Center ein, und klicken Sie dann auf **weiter**.

2. Überprüfen Sie die angezeigten Produktdetails, die Anzahl der Lizenzen und die Abonnementdauer. Wenn Sie bereit sind, wählen Sie **einlösen** aus, um das Abonnement zu aktivieren.

   Das neu aktivierte Abonnement steht auf der Registerkarte " **Produkte** " im M365 Admin Center zur Verfügung.

## <a name="licensing-scenarios"></a>Lizenzierungs Szenarios

In diesem Abschnitt werden einige Beispiele für Lizenzierungs Szenarien für Abonnements beschrieben, die mit Azure und cloudprodukten verknüpft sind. In diesen Szenarien erfahren Sie, wie Abonnement Enddaten berechnet werden.

### <a name="scenario-1"></a>Szenario 1

Wenn Sie 100-Lizenzen für ein Abonnement aktiviert haben und Sie versuchen, das Enddatum mithilfe eines Schlüssels mit 25 Lizenzen zu erweitern, wird das Enddatum um drei (3) Monate für das Abonnement mit 100 Lizenzen verlängert.

### <a name="scenario-2"></a>Szenario 2

Wenn Sie 100-Lizenzen für ein Abonnement aktiv haben und Sie versuchen, das Enddatum mithilfe eines Schlüssels mit 100 Lizenzen zu erweitern, wird das Enddatum um 12 Monate verlängert.

### <a name="scenario-3"></a>Szenario 3

Wenn Sie über fünf (5) Lizenzen verfügen, die für ein Abonnement aktiv sind, und Sie versuchen, das Enddatum mithilfe eines Schlüssels mit 10 Lizenzen zu erweitern, wird das Enddatum um 24 Monate für das Abonnement mit fünf (5) Lizenzen (oder Arbeitsplätzen) verlängert.

### <a name="scenario-4"></a>Szenario 4

Wenn Sie versuchen, das Enddatum mithilfe eines Schlüssels mit mehr als 3X aktiven Lizenzen/Arbeitsplätzen zu erweitern, wird die folgende Fehlermeldung angezeigt. Diese Meldung ist Entwurfs bedingt, da das Enddatum des Abonnements um maximal 3 Jahre verlängert werden kann.
